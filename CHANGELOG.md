# Registro de cambios

Este archivo documenta cambios significativos de **Voces que Restauran**.

El proyecto utiliza una estructura inspirada en *Keep a Changelog*, adaptada a su fase pedagógica y de prototipado.

---

## Convenciones

Las categorías posibles son:

- **Added:** contenido o funciones nuevas;
- **Changed:** modificaciones importantes;
- **Fixed:** correcciones;
- **Removed:** elementos eliminados;
- **Security:** privacidad, protección o seguridad;
- **Accessibility:** mejoras o correcciones de accesibilidad;
- **Deprecated:** elementos que serán retirados.

Las versiones pueden utilizar:

```text
MAJOR.MINOR.PATCH
```

Ejemplo:

```text
1.2.0
```

Durante la preparación inicial puede mantenerse una sección:

```text
Unreleased
```

## Changed

- El proyecto dejó de plantear la construcción inmediata de una aplicación.
- La unidad didáctica se definió como etapa previa de cocreación.
- El alcance tecnológico inicial se limitó a un recurso web estático.
- El componente de inglés se definió con funciones comunicativas A1–A2.
- La evaluación se orientó hacia procesos, evidencias y revisión.
- Los prototipos estudiantiles se definieron como insumos y no como productos institucionales definitivos.

## Security

- Se prohibió la recopilación de datos personales.
- Se descartaron cuentas y autenticación.
- Se descartó el almacenamiento local y remoto.
- Se prohibieron formularios de relatos y conflictos.
- Se estableció el uso de casos ficticios.
- Se prohibió el análisis automatizado de emociones.
- Se descartaron recomendaciones disciplinarias automáticas.
- Se definieron criterios para proteger productos estudiantiles.

## Accessibility

- Se estableció navegación mediante teclado.
- Se definió foco visible.
- Se priorizó HTML semántico.
- Se requirieron alternativas desconectadas.
- Se establecieron recursos imprimibles.
- Se definió compatibilidad con móviles y Chromebook.
- Se estableció que la información no dependa únicamente del color.

- recursos imprimibles;
- avisos de privacidad;

- cuentas;
- almacenamiento;
- formularios con envío;
- analítica;
- conexión directa con IA;
- gestión de casos;
- datos de estudiantes.

## Criterios de registro

Debe registrarse un cambio cuando:

- modifica el propósito;
- cambia la secuencia;
- añade una función;
- altera privacidad;
- modifica accesibilidad;
- cambia la arquitectura;
- afecta documentos relacionados;
- corrige un error significativo;
- elimina contenido o funciones.

No es necesario registrar:

- correcciones menores de puntuación;
- ajustes de formato sin impacto;
- cambios internos temporales no publicados.
